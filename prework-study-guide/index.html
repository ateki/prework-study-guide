Git

  • git status: checks what branch we are currently on. Remember we don't want to add our files to main. We will save our work in progress to a new feature branch.
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: To add code to current working branch. Add modifications in current working branch to staging area.
    -A flag indicates want to add all changes. The staging area informs Git that the changes will be added in the next commit.
  • git commit -m "comment re what is being added": commit captures a snapshot of currently staged changes.
  • git pull origin main: Before push changes to repo. check our local branch is in sync with base branch in GitHub.
    So pull down any changes from base branch. The pull receives modifications ofa branch in to local environment.
    Origin indicates source of the pull will be the GitHub rep.
    IMPORTANT: to always doa git pull tomake sure have latest version of project on local computer before you start to work.
  • git push origin feature-branch-name: Once local branch is synched with remote base branch, push changes to remote branch.
    Push changes we have locally to our remote GitHub branch. When using push can either create new remote branch or add new changes to existing branch.
  • Compare & Pull Request: Go to GitHub repo in browser, select feature branch from dropdown menu. Select the compare & pull request.
  • (use "git add ..." to update what will be committed)
  • (use "git restore ..." to discard changes in working directory)
